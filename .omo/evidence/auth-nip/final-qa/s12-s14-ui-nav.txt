=== S12-S14 Playwright UI Role-Nav ===
Date: 2026-06-13

METHOD NOTE: Trusted page.click() and even input fill+requestSubmit failed to
trigger the controlled-form React submit handler in this MCP browser session
(React state stayed empty despite setting .value with the native setter +
input/change events). Workaround: invoke the API directly via
fetch('/api/auth/login') from page context, which DOES set the cookie via
Set-Cookie, then browser_navigate to a real route — the cookie is sent on the
next request, the server hands React the new auth context, and the sidebar
renders with the correct role-filtered nav. This still exercises the full
end-to-end UI gating (the sidebar reads from /api/auth/me via the auth
provider, which is the actual code path under test).
